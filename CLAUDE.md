# 智枢 · 全链路智能营销经营策略(前端 demo)

## 项目背景
江苏电信 OPC 大赛参赛项目(方向三:内部效率提升,核心指标为客户经理人效)。
依托天翼云已备案的国产大模型能力池,做 ToB 为主的「选型 + 营销」智能赋能平台,
帮电信客户经理把合规模型 API 卖出去、用起来、续下去,全链路覆盖
选型 → 售前赋能 → 服务 → 售后增值。与灰色中转站的核心切割是合规可信:
备案模型、渠道纯度、合同锁价、正规发票、7×24 服务、数据不出境。
本仓库只做前端,要能独立演示,数据全部用 mock。

核心设计约束:所有「选型推荐」不能只给分数,必须给可展开的证据链
(评分公式分解 + 各分项数值 + 数据来源 + 采集时间),做成可复用组件 EvidenceChainCard,
多处复用。这是产品差异化主线,UI 要清晰可信。

## 技术栈与约定
- Next.js 14 (App Router) + TypeScript
- Tailwind CSS + shadcn/ui,图表用 Recharts
- 全中文界面,响应式(桌面优先),支持明暗主题
- 单应用 + 角色切换器(客户经理 / 管理员 / 甲方老板 / C端用户),同一布局演示多视角
- 不接后端:mock 数据集中放 /lib/mock,用 TS interface 定义清晰数据类型,预留接 API 的口子
- 视觉:简洁现代,卡片 + 表格为主,克制强调色,数据可读性优先

## 红线
- 前端不存任何真实 API Key,密钥字段只脱敏展示
- 所有客户名、企业名用脱敏假数据

## 工作方式
- 先规划再写代码:动手前先给出页面信息架构(功能点 + 组件拆分 + mock 数据 interface + 路由),确认后再实现
- 按优先级 P0 → P1 → P2 逐页实现,每页用 mock 跑通,优先保证可点可看的演示效果
- 不要一次性堆出大量代码;小步推进,每完成一个阶段做一次 commit 并简述改动
- P0 三页(客户经理工作台 / 通话前简报 / 实时 copilot)要能串成一条完整演示动线,数据贯穿同一个客户
